What are Constraints?

Constraints are column-level rules in a database, that enforce data integrity and consistency by restricting the type, format and permissible values store in a column.

( or )

Constraints are column-level rules, that controls how data is stored in a database to keep it accurate and consistent.

v1.3-unique

Unique Constraint - no repetition

It is a Constraint, which can't accept repeated / duplicated values.

( or )

A Constraint, which prevents duplicate values in a table column, called as Unique constraint.

( or )

An Unique Constraint, ensures that all values in a column are distinct and prevents duplicate entries.

Syntax :

UNIQUE

Let's take an example,

Student Table
ID Student_Name Degree City
1 Ameen B.Tech NULL
2 Dhana B.Tech DEL
3 Imran M.Tech CCU
4 Somu M.Sc. BLR
5 Dileep MCA HYD

In the above example, the 'City' column is defined with a UNIQUE constraint, meaning it does not allow duplicate values, as each city has a distinct code. If a record is inserted with a city that already exists, an error will occur, ensuring all city values remain unique.

Note :

A UNIQUE constraint can allow NULL values.

Reason : NULL is treated as unknown, so multiple NULL values are usually allowed ( depends on the database system ).

v1.2-Notnull

Null Constraint - Punish empty

It is a Constraint, which can't accept Empty / Null values.

( or )

A Constraint, whose attribute values can't be Null.

Syntax :

NOT NULL

From the previous example,

Student Table
ID Student_Name Degree City
1 Ameen B.Tech NULL
2 Dhana B.Tech DEL
3 Imran M.Tech CCU
4 Somu M.Sc. BLR
5 Dileep MCA HYD

In the above example, the 'Degree' column is defined with a NOT NULL constraint, meaning it does not allow empty values. If a record is inserted without specifying a degree, an error will occur, ensuring that every entry has a valid degree value.

Note :

NOT NULL constraint can allow duplicate values.

v1.3-check

Check Constraint - restriction

An Extra Validation process assigned to the column with the help of conditions, if the conditions are satisfy it returns true then accept the value else not accept value.

( or )

A Constraint, which allows you to specify a condition that must be met for the data to be inserted or updated in a column.

( or )

A Check constraint is a rule that validates whether the data meets a specific condition.

Syntax :

CHECK( condition )

For example,

In the Student Table, the 'City' column is defined with a constraint that ensures it accepts only city codes of exactly 3 characters (such as HYD, BLR), preventing any values that do not meet this length requirement.

v1.3-Default

Default - automatic

A Default constraint assigns a predefined value to a column when no value is provided during insertion. It ensures that the column always contains a valid value even if the user does not specify one.

( or )

It is a constraint, which assigns a default value to a column when no value is provided during insertion.

Syntax :

DEFAULT

v1.3-Entity

Primary Key - Identitfier

Primary Key also as an Entitiy Integrity constraint, It is a constraint that combines UNIQUE and NOT NULL properties to uniquely identify each row in a table and ensures the column value is left null.

Syntax :

PRIMARY KEY

Characteristics :

  • Primary Key cannot accpet Repeated & Null values ( or ) No attributes participating in primary key is allowed to accept Duplicate and Null values
  • It is not Mandatory but highly recommanded - as it helps in joining with another table.
  • It is immutable — once created, the value of a primary key cannot be changed.

Lets take an example from 'Student Table',

Student Table
ID Student_Name Degree City
1 Ameen B.Tech NULL
2 Dhana B.Tech DEL
3 Imran M.Tech CCU
4 Somu M.Sc. BLR
5 Dileep MCA HYD

In the above example, the 'ID' column is defined as the Primary Key, meaning it uniquely identifies each row. It does not allow duplicate or NULL values, ensuring every student has a distinct identifier.

v1.3-Refer

Foreign Key - Reference

Foreign Key also as Referential Integrity constraint, It is a constraint used to estalish a relation between 2 tables by referencing the primary key of another table.

Syntax :

FOREIGN KEY

Characteristics :

  • Foreign Key can accpet duplicate & null values.
  • A table can have multiple foreign key columns.
  • A foreign key exists in the child table and references the primary key in the parent table.
